Screen Definition:
• Current password field
• New password field
• Confirm password field
• Save changes button
• Cancel button

User Stories:
As a user, I want to update my password when I want so that I can maintain the security of my account and protect my account from unauthorized access.

Acceptance Criteria:
• The current password field input should match the Database. If it does not match, the system should display "Password does not match".
• The new password should meet the system criteria (length at least 8, 1 uppercase letter, 1 number) if it does not meet the criteria the system should display “Please match the criteria".
• All fields should be mandatory if any field is empty, system should display "Please fill all the required fields".
• The new password and confirm password should be the same; if not, the system should display "Password mismatch" but if match then the system should display "Password matched".
• The Save Changes button will save the changes to the DB if all fields are filled correctly; if anything is missing, the system should display "Unable to save changes".
• The Cancel button should cancel all changes and return to the previous page.
