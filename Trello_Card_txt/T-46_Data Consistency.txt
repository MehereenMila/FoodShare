User Stories:

As a system admin, I want the system to maintain consistent data across all modules so that users always see the same and correct information regardless of where they access it.

Acceptance Criteria:

• The system should ensure that all data ( food listings, user profiles, and donation records) remains the same across all modules and screens at all times.
• Any update made to data in one module should be immediately reflected in all other related modules within a few seconds (near real-time synchronization).
• The system should prevent conflicting updates when multiple users try to modify the same data at the same time.
• The database should enforce rules to avoid duplicate, missing, or mismatched records.
• Data retrieved from the system should always match the latest committed database state.
