Screen Definition:
• Assigned Task list table
• Task details view panel
• Accepted button
• Rejected button
• Status
• Refresh button

User Story:
As a Volunteer, I want to view my assigned tasks so that I know what deliveries I need to complete and also I want to update task status so that progress is tracked properly.

Acceptance Criteria:
• Only assigned tasks must be displayed to the logged-in volunteer.
• Each task must include task ID, food name, pickup location, delivery location, and current status.
• It must allow viewing detailed task information upon selection.
• The  empty state should show when no tasks are assigned.
• Volunteer should be able to accept or reject the task.
• Notifications should be sent as "Accepted Task" or "Pending Task" or "Rejected Task".
• The status should be updated based on volunteer action.
